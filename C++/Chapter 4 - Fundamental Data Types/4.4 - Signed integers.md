# 4.4 — Signed integers

## What is an integer?

An **integer** is an integral type that can represent positive and negative whole numbers, including 0.

Examples:
- -2
- -1
- 0
- 1
- 2

C++ provides **four primary fundamental integer types**:

| Type        | Minimum Size | Notes                                  |
|-------------|--------------|----------------------------------------|
| short       | 16 bits      |                                        |
| int         | 16 bits      | Typically 32 bits on modern systems    |
| long        | 32 bits      | 32 or 64 bits depending on platform    |
| long long   | 64 bits      |                                        |

> C++ guarantees minimum sizes, not exact sizes.

You can check the size on your machine using `sizeof`.

---

## Defining signed integers

Preferred style:

```cpp
short s;
int i;
long l;
long long ll;
```

These also work but are discouraged:

```cpp
short int
long int
long long int
```

The `signed` keyword is optional because integers are signed by default:

```cpp
signed short ss;
signed int si;
signed long sl;
signed long long sll;
```

Prefer the shorter forms without `signed` or `int`.

---

## What does "signed" mean?

A number’s **sign** tells us whether it is:
- positive
- negative
- zero

By default, integers in C++ are **signed**, meaning they can store both positive and negative values.

---

## Signed integer ranges

A variable with **n bits** can hold:

2ⁿ possible values.

For signed integers (two’s complement representation), the range is:

-(2ⁿ⁻¹) to (2ⁿ⁻¹) - 1

### Common ranges

| Size            | Range |
|-----------------|-------|
| 8-bit signed    | -128 to 127 |
| 16-bit signed   | -32,768 to 32,767 |
| 32-bit signed   | -2,147,483,648 to 2,147,483,647 |
| 64-bit signed   | -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807 |

C++20 requires **two’s complement** representation.

Older standards allowed:
- sign-magnitude
- one’s complement

Those are now historical relics.

---

## Overflow

If a value exceeds the representable range of a signed integer, the behavior is **undefined**.

Example:

```cpp
#include <iostream>

int main()
{
    int x { 2'147'483'647 }; // max 32-bit signed int
    std::cout << x << '\n';

    x = x + 1; // overflow (undefined behavior)
    std::cout << x << '\n';

    return 0;
}
```

On many machines, this prints:

```
2147483647
-2147483648
```

But because this is **undefined behavior**, the result is not guaranteed.

Overflow causes loss of information and should be avoided.

If you suspect values may exceed a type’s range, use a larger type.

---

## Integer division

When dividing two integers:

- The result is always an integer.
- The fractional part is **discarded**, not rounded.

Example:

```cpp
std::cout << 20 / 4;  // 5
std::cout << 8 / 5;   // 1
```

Why?

8 / 5 = 1.6  
The 0.6 is dropped.

Similarly:

```
-8 / 5 = -1
```

C++ truncates toward zero.

Be careful: integer division silently removes fractional parts.

## Key Takeaways

- C++ guarantees minimum sizes, not exact sizes.
- Signed integers use two’s complement.
- Overflow on signed integers is undefined behavior.
- Integer division discards fractional parts.
- When in doubt, choose a type with a larger range.
