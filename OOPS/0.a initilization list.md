# What?
A way to initialize members BEFORE the constructor body runs

# Syntax
```cpp fold title:'initializer list'
class A {
    int x, y;

public:
    A(int a, int b) : x(a), y(b) {
        // constructor body
    }
};
```

# Why is better?
- why not like this

```cpp fold title:'old constructor'
A(int a, int b) {
    x = a; // assignment
    y = b;
}
```

Initializer list → direct initialization
Inside body → assignment after default initialization

# Why Matters?

## Effeciency
```cpp fold title:'string class'
class A {
    string s;

public:
    A(string str) : s(str) {}  // ✅ direct

    // vs

    A(string str) {
        s = str; // extra work
    }
};
```

1. s default constructed
2. Then assigned → waste
## Required in some cases
### Const members

```cpp fold title:'COnst Members'
class A {
    const int x;

public:
    A(int a) : x(a) {} // ✅ required
};
```

```cpp fold title:'fail'

A(int a) {
    x = a; // ERROR
}
```

> [!NOTE]
> This is will fail because `const` members once initilized cannt be re-initialzed

### Reference members
```cpp fold title:'Reference'
class A {
    int& ref;

public:
    A(int& r) : ref(r) {} // ✅ required
};
```

### Base class

```cpp fold title:'title'
class Base {
public:
    Base(int x) {}
};

class Derived : public Base {
public:
    Derived(int x) : Base(x) {} // ✅ required
};
```

### Member objects

```cpp fold title:'Member objects'
class B {
public:
    B(int x) {}
};

class A {
    B obj;

public:
    A(int x) : obj(x) {} // ✅ required
};
```

### Order of initialization

```cpp fold title:'Order'
class A {
    int x;
    int y;

public:
    A(int a, int b) : y(b), x(a) {}
};
```

- Actual order:
    - x initialized first
    - then y
- Order depends on declaration order, NOT initialization list order
