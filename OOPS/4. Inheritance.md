# Mode of inheritance

|               | Derived Class | Derived Class | Derived Class |
| :------------ | ------------- | ------------- | ------------- |
| Base Class    | **Private**   | **Public**    | **Public**    |
| **Private**   | Not Inherited | Not Inherited | Not Inherited |
| **Protected** | Private       | Protected     | Protected     |
| **Public**    | Private       | Protected     | Public        |

# Constructor 
- The main goal of the OOPs is to decrease repetitive code writing so. 
	- We don't actually need to rewrite the constructor of the derived class over again
- Here's an example

```C++ fold title:'Constructor inheritance' hl:1,8,16,19
class A{
	//some attributes
	public:
	A(int a,int b,int c){
		// setting these attribues
	}
};
class C{
	//some attributes
	public:
	A(int d,int e,int f){
		// setting these attribues
	}
};

class B : public A{
	//some more attributes
	public:
	B(int a,int b,int c,int d,int e,int f,int g) : 
				A(a,b,c), 
				C(d,e,f)
				{
		this->g = g;
	}
};

```

## `L-value` and `R-value`

| type    | properties                           |                examples                 |
| :------ | :----------------------------------- | :-------------------------------------: |
| L-value | Variable having memory locations     |              `int a = 10`               |
| R-value | Variable not having memory locations | `5`,`int &a = b //something like alias` |
## Const Variable
- **Const variable**

```c++ fold title:'Const variable' 
const int p =19;
p = 10; ❌ -> error
```

- **Const pointer cases**
- *Pointer to const data*
```c++ fold title:'Pointer to const data' 
const int *p= &x;  //cannot change *p
int const *p= &x;  //Same as above
*p = 10; ❌ -> error
```
- *Const pointer*
	- Like the java-script one the address is fixed but not the actual data
```c++ fold title:'const Pointer' 
int* const p = &x;   // cannot change p
*p = 10; ✅

// one example
int* const arr = new int[10];
arr[0] = 19; // valid 
```

- *Const pointer to const data*

```c++ fold title:'Const Pointer to Const Data'
const int* const p = &x; // cannot change *p && // cannot change p
```

> [!NOTE]
> ==const before `*` makes the pointer `data` constant==
> ==To make make the location constant we use it after the `*`==

- **Const function arguments**
- *Prevents modification inside function*
```C++ fold title:'const inside a function'
void func(const int a) {  
	// a cannot be changed  
}
```
    
- *With references (important)*
```C++ fold title:'References'
void func(const int& a) {  
	// avoids copy + protects value  
}
``` 

- **Const return type**
- Prevents modification of returned value (mostly useful with references/pointers)
```C++ fold title:'return type const'
const int& getVal();
```
    
- **Const member function**
- *Guarantees object state won’t change*
```C++ fold title:'const w.r.t classes'
class A {  
public:  
	int x;  
	int get() const {  
		return x; // cannot modify x here  
	}  
	//*Guarantees object state won’t change*
};
```    
- **Const objects**
    - Can only call const member functions
    ```C++ fold title:'const objects'
const A obj;  
obj.get();     // ✅  
// obj.set();  // ❌ if not const
    ```
    
- **Const with arrays / STL**
```C++ fold title:'Arrays and all'
void print(const vector<int>& v) {  
	// cannot modify v  
}
```

## `mutable` keyword

- mutable allows a class member to be modified even inside a const object or const member function
- only used for debugging purposes

```cpp fold title:'mutable keyword'

class A {
public:
    mutable int x;

    void show() const {
        x = 10;
    }
};
```
- cannot be used the local variables
- and function arguments

- it is exception to the `const` keyword
