# 📘 Memory Layout of a C/C++ Program (Brief Explanation)
---

## 1. TEXT Segment (Code)

- Contains **compiled program instructions**
- This is your actual code (`main()`, functions, etc.)
- Usually **read-only**

👉 Example:
```c
int main() { return 0; }
```

---

## 2. RODATA (Read-Only Data)
- Stores **constants and string literals**
- Cannot be modified

👉 Example:

```c
printf("Hello"); // "Hello" stored here
```

---

## 3. DATA Segment (Initialized Data)

- Stores **global and static variables that are initialized**

👉 Example:

```c
int x = 10; // goes here
```

---

## 4. BSS Segment (Uninitialized Data)

- Stores **global and static variables that are NOT initialized**
- Automatically initialized to **0**

👉 Example:

```c
int x; // goes here (value = 0 by default)
```

---

## 5. HEAP

- Used for **dynamic memory allocation**
- Managed manually (`malloc`, `new`, `free`, `delete`)
- **Grows upward**

👉 Example:

```c
int* p = new int;
```

---

## 6. STACK

- Stores:
    - Function calls
    - Local variables
    - Parameters
- Managed automatically
- **Grows downward**

👉 Example:

```c
void func() {
    int x = 5; // stored in stack
}
```

---

## ⚠️ Important Behavior

- **Stack grows downward ⬇**
- **Heap grows upward ⬆**
- If they collide → ❌ crash (stack overflow / heap overflow)

---

## 🧠 Mental Model

```
[ STACK   ]  ↓ (function calls, locals)
[ HEAP    ]  ↑ (dynamic memory)
[ BSS     ]  (uninitialized globals)
[ DATA    ]  (initialized globals)
[ RODATA  ]  (constants)
[ TEXT    ]  (code)
```

---

## Key Takeaways

- Local variables → **Stack**
- Global/static → **Data / BSS**
- Dynamic memory → **Heap**
- Code → **Text segment**
