###  Operators That Cannot Be Overloaded

| Operator    | Meaning               |
| ----------- | --------------------- |
| **::**      | Scope resolution      |
| **.**       | Member access         |
| **.***      | Pointer-to-member     |
| **?:**      | Ternary operator      |
| **sizeof**  | Size operator         |

| Rule                            | Description                  |
| ------------------------------- | ---------------------------- |

# Runtime Polymorphism 
## Problem : 'Early binding'
- (also called **static binding** or **compile-time binding**) 
- mechanism where the compiler resolves method calls, variable types, and function references **at compile time**
- before the program runs.

## Context

```C++ fold title:'Example'
class Animal {
public:
    void speak() {
        cout << "Animal speaks" << endl;
    }
};

class Dog : public Animal {
public:
    void speak() {
        cout << "Dog barks" << endl;
    }
};

int main() {
    Animal* a = new Dog();
    a->speak();  // Output: "Animal speaks"  ← Early binding!
}
```

>[!Note]
>Here, since `speak()` is **not virtual**, the compiler binds the call to `Animal::speak()` based on the **declared type** (`Animal*`), not the actual object (`Dog`). This is early binding

## Virtual Keyword 
- The `virtual` keyword delays the compiler's decision until **runtime**.

```C++ fold title:'Example'
class Animal {
public:
    virtual void speak() {
        cout << "Animal speaks" << endl;
    }
};

class Dog : public Animal {
public:
    void speak() {
        cout << "Dog barks" << endl;
    }
};

int main() {
    Animal* a = new Dog();
    a->speak();  // Output: "Dog Barks"  
				//← Early binding issue fixed
}
```

|                             | Decision Time                           |
| --------------------------- | --------------------------------------- |
| ~={pink}Without=~ `virtual` | Compiler decides **now** (compile time) |
| ~={pink}With=~ `virtual`    | Compiler decides **later** (runtime)    |

## `Override`

- The `override` keyword is used in a child class to **explicitly tell the compiler** that you are intentionally overriding a `virtual` function from the parent class.
- Synopsis
	- `override` = "Hey compiler, I am intentionally replacing my parent's virtual function. Please verify that."

```C++ fold title:'without override'
class Animal {
public:
    virtual void speak() { cout << "Animal"; }
};

class Dog : public Animal {
public:
    void speak() { 
    cout << "Dog"; // works, but compiler won't verify
    }  
};
```

```C++ fold title:'with override'
class Animal {
public:
    virtual void speak() { cout << "Animal"; }
};

class Dog : public Animal {
public:
    void speak() override{ 
    cout << "Dog"; // compiler verifies this
    }  
};
```

>[!quote] **`virtual` gives permission to override. `override` confirms you are doing it correctly.**

## More on UP-CASTING & DOWN-CASTING
- These are about **converting object references** between parent and child types in inheritance.

### UP - CASTING

>[!info] **Treating a child class object as a parent class type.**
- Goes **UP** the inheritance hierarchy (Child → Parent)
- **Automatic / Implicit** — compiler does it safely
- **Always safe** — every child IS a parent

```c++
class Animal {
public:
    virtual void speak() { cout << "Animal"; }
};

class Dog : public Animal {
public:
    void speak() override { cout << "Dog"; }
    void fetch() { cout << "Fetching!"; }
};

int main() {
    Dog d;
    Animal* a = &d;   // ✅ Upcasting — automatic, no casting needed
    a->speak();       // Output: "Dog"  (virtual still works!)
}
```
### What You LOSE with Upcasting
```C++
a->fetch();   // ❌ ERROR! — fetch() is not in Animal
```
You **lose access** to child-specific methods because the compiler only sees the **parent type**.
### Down-casting
> **Treating a parent class pointer/reference BACK as a child class type.**

- Goes **DOWN** the hierarchy (Parent → Child)
- **Manual / Explicit** — you must do it yourself
- **Risky** — not always safe

```cpp
int main() {
    Animal* a = new Dog();     // Upcasted first

    Dog* d = (Dog*)a;          // Downcasting — manual cast
    d->fetch();                // ✅ Now you can access fetch()
}
```
#### Safe Downcasting with `dynamic_cast`

Raw casting is dangerous. Use `dynamic_cast` to cast **safely**:

```cpp
int main() {
    Animal* a = new Dog();

    Dog* d = dynamic_cast<Dog*>(a);  // Safe downcast

    if (d != nullptr) {
        d->fetch();   // ✅ Safe to use
    } else {
        cout << "Cast failed!";  // Won't crash
    }
}
```

### Why `dynamic_cast` is Safer

```C++
Animal* a = new Animal();         // NOT a Dog!

Dog* d = dynamic_cast<Dog*>(a);   // returns nullptr — safe ✅
Dog* d = (Dog*)a;                 // Undefined behavior — dangerous ❌
```
### Visual Picture

```
        Animal         ← Parent (Base)
          |
         Dog           ← Child (Derived)


Upcasting:    Dog → Animal   (going UP)   ✅ Automatic, Safe
Downcasting:  Animal → Dog   (going DOWN) ⚠️ Manual, Risky
```
### Side by Side Comparison

|             | Upcasting           | Downcasting           |
| ----------- | ------------------- | --------------------- |
| Direction   | Child → Parent      | Parent → Child        |
| How         | Automatic           | Manual                |
| Safety      | Always safe         | Can fail              |
| Access      | Loses child methods | Regains child methods |
| Cast needed | No                  | Yes (`dynamic_cast`)  |

### Connecting Everything So Far

```
virtual     →  delay decision to runtime
override    →  confirm you are replacing virtual function
upcasting   →  treat Dog AS Animal (safe, automatic)
downcasting →  treat Animal BACK AS Dog (risky, manual)
```

### One Line Summary
>[!summary] **Upcasting is widening your view (safe). Downcasting is narrowing your view (risky — verify first).**


## `Final` Keyword 

- used to stop further over-riding or further inheritancce 
	- synopsis :- > **`final` = "This is the end. No one can go beyond this point."**
### Stopping inheritance 
```C++ fold title:'Final keyword'
class Animal final {   // No one can inherit from Animal
public:
    void speak() { 
	    cout << "Animal"; 
	}
};

class Dog : public Animal {   // ❌ COMPILE ERROR!
};
```
>[!quote] Once a class is marked `final`, **no child class can inherit** from it.

### On Virtual Function
- prevents over-riding of the `virtual` function

```C++ fold title:'Virtual function and final'
class Animal { 
	public: 
		virtual void speak() { 
			cout << "Animal"; 
		} 
	}; 

class Dog : public Animal { 
	public: void speak() final { 
		cout << "Dog"; 
	} // No further overriding allowed 
}; 
class Labrador : public Dog { 
	public: void speak() override { 
		cout << "Labrador"; 
} // ❌ COMPILE ERROR! };
```
